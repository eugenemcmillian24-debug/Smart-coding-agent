# Smart Full-Stack Coding Agent

An interactive React app that automates software development by generating code, reviewing it for quality and security, and executing it in a sandboxed environment.

## Data Contract

### Inputs

| Field | Type | Description |
|-------|------|-------------|
| `task_description` | string | Clear description of the feature or app to build |
| `target_stack` | string | Technology stack (e.g. "Node.js + Express backend, React frontend") |
| `repository_url` | string | Destination Git repository for committing generated code |
| `execution_environment` | ExecutionEnvironment | Docker image, CPU/memory limits, timeout |
| `config` | Config | Coding style, security policies, test preferences, lint rules |

#### Config Fields
- `coding_style`: "airbnb" | "google" | "standard" | "prettier" | "none"
- `security_policies`: string[] — e.g. `["no-eval", "sanitize-inputs", "strict-csp"]`
- `test_preferences.framework`: "jest" | "mocha" | "vitest" | "pytest" | "unittest"
- `test_preferences.include_integration_tests`: boolean
- `test_preferences.coverage_threshold`: number (0-100)
- `lint_rules`: string[] — additional lint rules

#### ExecutionEnvironment Fields
- `docker_image`: e.g. "node:18-alpine", "python:3.11-slim"
- `resource_limits.cpu`: e.g. "1", "500m"
- `resource_limits.memory`: e.g. "512Mi", "2Gi"
- `resource_limits.timeout_seconds`: number

### Outputs

| Field | Type | Description |
|-------|------|-------------|
| `generated_code` | SourceFile[] | Collection of source files produced by the Code Generator Agent |
| `review_report` | ReviewReport | Structured report with quality score, security issues, suggestions, style violations, test coverage |
| `execution_results` | ExecutionResults | Sandbox outcome with success flag, logs, exit code, artifacts |
| `summary` | Summary | Human-readable status, next steps, file/issue counts, execution time |
| `deployment_info` | DeploymentInfo | Commit SHA, URL, branch, status, error tracking |

#### ReviewReport Fields
- `quality_score`: number (0-100)
- `overall_status`: "pass" | "warn" | "fail"
- `security_issues[]`: severity, file, line, description, cwe_id, remediation
- `suggestions[]`: improvement recommendations
- `style_violations[]`: style guide violations
- `test_coverage`: number (0-100)

#### ExecutionResults Fields
- `success`: boolean
- `logs[]`: build and test output lines
- `error_message`: error details if failed
- `duration_ms`: total execution time
- `exit_code`: process exit code
- `artifacts[]`: output directories/files produced

#### DeploymentInfo Fields
- `commit_sha`: Git commit hash
- `url`: deployment URL
- `branch`: target branch
- `status`: "deployed" | "skipped" | "failed" | "pending"
- `deployed_at`: ISO timestamp
- `error`: failure reason

## Features

- **4-step wizard** for task input with structured config and environment fields
- **AI-powered generation** — code and review reports generated by AI service
- **Inline code editor** — edit generated files before execution or deployment
- **Live sandbox execution** — "Run in Sandbox" emits project to this chat for real execution
- **GitHub deployment** — "Push to GitHub" sends code to configured repository

## Tech Stack

- React + TypeScript
- Tailwind CSS + DaisyUI
- Zenfox platform services (data persistence, AI completion, events)

## File Structure

```
src/
  types.ts              # Data contract interfaces
  App.tsx               # Main orchestrator with AI generation pipeline
  components/
    TaskInput.tsx       # 4-step wizard form
    RunDashboard.tsx    # Detail view with tabs + editor + deploy buttons
    RunHistory.tsx      # Run list with status badges
```

## Usage

1. Open the app from your Zenfox Dock
2. Click "New Run" and fill the 4-step wizard
3. Review generated code, edit if needed
4. Click "Run in Sandbox" to execute live in this chat
5. Click "Push to GitHub" to deploy to your repo
